# StoryAI 진행 현황 / 다음 작업 (2026-07-13 기준)

우리 아이 사진 → AI가 개인화 동화책(PDF)/영상 만들어주는 서비스.
경쟁 벤치마크: talecoco.com (파스텔 반실사, 페이지 단가제).

---

## ✅ 지금까지 완료 (깃허브 main에 반영됨)

- **AI 파이프라인**: 사진 업로드 → 캐릭터 시트(평상복+주제의상) → 페이지별 삽화 → 가로 PDF. Gemini(나노바나나)로 스토리 글 + 그림 모두 생성.
- **캐릭터 일관성**: 참조 이미지로 같은 아이 얼굴 유지, 표지=실제 옷 → 중간에 주제 의상으로 변신 연출.
- **연령별 텍스트**: 3~4 / 5~6 / 7~8 / 9~10세 (길이·의성어 조절).
- **헌정 메시지** → 헌정 페이지, **스토리 방향**(선택) 입력 반영.
- **사진 크롭**: 부모+아이 사진에서 아이만 잘라서 업로드.
- **그림 스타일 5종**(동화/스케치/색연필/수채화/카툰) + **주제 11종**(공주/왕자/용사/모험/동물친구/우주/공룡/바다/숲속/정글/해적) + 주제별 의상.
- **손글씨 폰트**(제목 Jua, 본문 Gaegu), 좌/우/상하/풀블리드 레이아웃 혼합.
- **가격 표시 숨김**(결제 단계에서 노출 예정).
- **[오늘] 그림 꽉 채우기**: 여백 제거(cover) + **가로(3:2) 그림 생성** → 페이지 끝까지 그림이 참. (검증: 1264×848 landscape 확인)
- 24페이지 / 36페이지 둘 다 생성 가능, 삽화 장수 제한 없음(전체 생성).

## ⏳ 내일 진행할 작업 (우선순위 순)

### 1) [최우선] 미리보기 2단계 구조  ← 비용 폭탄 방지의 핵심
- **미리보기 6장만 먼저 생성**: 표지 + 헌정페이지 + 삽화 4장 (헌정 없으면 표지 + 삽화 4장 = 5장).
- 병렬 생성으로 미리보기 시간 단축(약 1~1.5분 목표).
- 미리보기 비용 ≈ 330원/건 (전체 1,600원 대비 1/5). **이탈해도 미리보기 값만 부담.**
- 나머지 전체 페이지는 **구매/확인 이후**에만 생성.

### 2) 미리보기 후 구매 선택 화면
- 미리보기 확인 → **PDF로 받기 / 실물 책으로 받기** 선택.
- 결제는 아직 안 붙임(설계상 나중). 실물은 배송 정보만 수집 → 사장님이 인쇄소로 수동 발주.

### 3) 완성본 전송 (외부 계정 필요 → 방식 결정해야 함)
- **이메일 우선 추천**: Gmail SMTP / SendGrid 무료로 바로 가능.
- **카카오톡**: 사업자등록 + 카카오 채널 + 알림톡 심사 필요 → 개인은 당장 어려움. 사업자등록 후 추가.
- 👉 **결정 필요: 이메일로 시작할지 확정.**

### 4) 배포 (그 다음 단계)
- 프론트 → **Netlify(무료)**, 백엔드+DB → **Railway(월 몇 천원, 추천)**. AWS는 지금 불필요(과함).
- 배포 전 필수: **영구 DB(H2→PostgreSQL) 교체**, **비용 상한/요청 제한**(악용 방지), 도메인+HTTPS, 개인정보 약관.
- 도메인: 가비아 구매 예정.

## 🐛 알려진 버그 (내일 수정)
- **주인공 1명만 넣었는데 삽화에 2명이 나옴** (예: page-3).
  - 확인된 것: page-3는 새로 생성된 파일이고, 캐릭터 시트도 1명분만 생성됨(참조=1명). 그런데 그림엔 2명.
  - 유력 원인: **스토리/장면(scene) 생성 시 AI가 친구·형제를 임의로 추가** → 장면 문장에 두 아이가 언급되어 그림 모델이 없는 2번째 아이를 그림.
  - 수정: (1) 스토리 프롬프트에 "등장인물은 지정된 N명뿐, 다른 아이 발명 금지" 강제, (2) 삽화 프롬프트에 "참조된 N명만 그리고 다른 사람 추가 금지" 강제, (3) page-3의 실제 scene 문장 로그/DB로 검증.
  - 관련 파일: `ai/story/GeminiStoryGenerator.java`, `ai/image/GeminiImageGenerator.java`
  - 참고: dev는 H2 리셋으로 jobId가 1로 재사용돼 `gen/1/` 폴더에 옛 파일이 섞임 → 조사 시 타임스탬프로 신규분 구분할 것.

## 💰 비용 메모

## 🖥️ 로컬 실행 (개발 PC는 가상화 꺼져 Docker 불가 → H2 dev 프로파일)
- 백엔드: `backend/` 에서 `gradlew.bat bootRun --args=--spring.profiles.active=dev` (GEMINI_API_KEY 환경변수 주입)
- 프론트: `frontend/` 에서 `npm run dev` (→ :5173, /api는 :8080 프록시)
- 주의: H2 메모리라 백엔드 재시작 시 업로드/주문 데이터 초기화됨(사진 파일은 디스크에 남음).
